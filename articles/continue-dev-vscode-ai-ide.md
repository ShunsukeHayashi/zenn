こんにちは！フリーランスエンジニア7年目のハヤシシュンスケです。

「CursorやGitHub Copilotは高い…でもAI開発の恩恵は受けたい」

そう感じているエンジニア、実は多いんじゃないでしょうか。私もそのひとりでした。月額$20のCursorを惰性で契約しながら、「これ、本当に元取れてるのか？」と自問する日々。

そんなときに出会ったのが **Continue.dev** です。

オープンソース、VSCode拡張、自分でモデルを選べる——このツールが、私の開発環境を劇的に変えました。今日はその全貌を、失敗談も含めて正直にシェアします。

> **📝 注記**: 本記事は2025年3月時点の情報に基づいています。Continue.devは活発に開発が続いており、最新情報は公式ドキュメントも合わせてご確認ください。

## Continue.devとの出会い：「無料でここまでできるのか」

きっかけはX（旧Twitter）のタイムラインでした。「CursorをVSCodeで再現できる」という投稿が目に入り、半信半疑でインストールしてみたのが3ヶ月前。

正直、最初は期待していませんでした。「無料のものって、結局中途半端なんでしょ」という先入観がありました。

でも実際に触ってみたら、想像をはるかに超えていました。

Continue.devは**オープンソースのAIコードアシスタント**です。VSCode（とJetBrains）の拡張機能として動作し、自分の好きなAIモデルを接続できます。Claude、GPT-4o、Gemini、さらにはローカルで動くOllamaまで対応。しかも完全無料でコア機能が使えます。

「AIコーディングツールの民主化」——使い始めて、そう思いました。

## やらかした失敗談3つ

でも、スムーズだったわけじゃありません。導入から最初の2週間、3つの大きな失敗をやらかしました。

### 失敗1：設定ファイルを「なんとなく」コピペした

インターネットで見つけた `config.json` をそのままコピペしたんです。見た目は動いているように見えて、実際には古いAPIキーの形式が混在していたせいで、コンテキストが正しく渡されていない状態が1週間続きました。

```json
// ❌ よくあるコピペミス：古い形式と新しい形式が混在
{
  "models": [
    {
      "title": "Claude",
      "provider": "anthropic",
      "model": "claude-opus-4-5",  // 古いモデル名
      "apiKey": "sk-ant-..."       // ここは正しい
    }
  ],
  "contextProviders": [
    {
      "name": "codebase",
      "params": { "nRetrievalResults": 10 }  // デフォルト値のまま
    }
  ]
}
```

**学んだこと**：公式ドキュメントを起点にして、1設定ずつ確認しながら組み立てる。コピペは「出発点」にするが、そのまま使わない。

### 失敗2：全部のコンテキストをAIに渡そうとした

「情報は多いほうが精度が上がるはず」という思い込みで、`@codebase` を毎回使い、プロジェクト全体のファイルをコンテキストに突っ込んでいました。

結果、レスポンスが遅い、回答がぼんやりする、APIのコストが跳ね上がる——という三重苦に。

**学んだこと**：「必要な情報だけ、ピンポイントで渡す」が原則。`@file` で特定ファイル、`@code` で特定関数を渡すほうが、回答の質がはるかに上がります。

### 失敗3：モデルを毎日変えてビジョンがブレた

「今日はClaude、明日はGPT-4o、明後日はGemini」と試しまくった結果、どのモデルが何が得意かわからなくなりました。比較するためのベースラインが固まっていないのに、次々と切り替えるのは意味がありません。

**学んだこと**：まず1つのモデルを2週間使い込んで「このモデルの癖」を把握する。その後に比較する。

## Before/After：数値で見る効果

失敗から学んで、正しく使えるようになった3ヶ月後の変化です。

| 指標 | Before（Cursor/Copilot） | After（Continue.dev） |
|---|---|---|
| 月額コスト | $20〜$40 | $0〜$15（API実費のみ） |
| コード補完の採用率 | 約35% | 約58% |
| 1日あたりの手作業タイピング量 | 約2,800行相当 | 約1,100行相当 |
| コードレビュー指摘数（週平均） | 12件 | 5件 |
| 新規機能実装の平均所要時間 | 4.2時間 | 2.5時間 |

コスト削減だけじゃなく、品質面でも明確に改善しました。特に「コードレビュー指摘数」の減少は想定外の収穫でした。AIと対話しながら書くことで、自然とセルフレビューの機会が増えるんです。

> コスト削減率: 約60〜70%、実装速度: 約40%向上

## 実践5ステップ：今日から始めるContinue.dev

### ステップ1：インストール

VSCodeの拡張機能検索で「Continue」と検索し、Continue.devの公式拡張をインストールします。

```bash
# コマンドラインからインストールする場合
code --install-extension Continue.continue
```

インストール完了後、サイドバーにContinueのアイコンが現れます。初回起動時にセットアップウィザードが走りますが、まずはスキップしてOKです。

### ステップ2：モデルの設定

`~/.continue/config.json` を開きます（コマンドパレットで「Continue: Open config.json」）。

最初はAnthropicのClaude推奨です。理由は「コードの文脈理解が深い」「長いコンテキストウィンドウ」「日本語の指示に対して素直に従う」の3点。

```json
{
  "models": [
    {
      "title": "Claude Sonnet（メイン）",
      "provider": "anthropic",
      "model": "claude-sonnet-4-6",
      "apiKey": "YOUR_ANTHROPIC_API_KEY"
    },
    {
      "title": "Gemini Flash（高速サブ）",
      "provider": "google",
      "model": "gemini-1.5-flash",
      "apiKey": "YOUR_GOOGLE_API_KEY"
    }
  ],
  "tabAutocompleteModel": {
    "title": "Codestral（補完専用）",
    "provider": "mistral",
    "model": "codestral-latest",
    "apiKey": "YOUR_MISTRAL_API_KEY"
  }
}
```

APIキーの取得先：
- Anthropic: [console.anthropic.com](https://console.anthropic.com)
- Google: [aistudio.google.com](https://aistudio.google.com)
- Mistral: [console.mistral.ai](https://console.mistral.ai)（Codestralは月30リクエストまで無料）

### ステップ3：コンテキストプロバイダーの設定

これがContinue.devの真骨頂です。どんな情報をAIに渡すかをカスタマイズできます。

```json
{
  "contextProviders": [
    {
      "name": "code",
      "params": {}
    },
    {
      "name": "docs",
      "params": {}
    },
    {
      "name": "diff",
      "params": {}
    },
    {
      "name": "terminal",
      "params": {}
    },
    {
      "name": "problems",
      "params": {}
    },
    {
      "name": "folder",
      "params": {}
    }
  ]
}
```

チャット欄で `@` を打つと利用可能なコンテキストが表示されます。よく使うのは：

- `@file` → 特定ファイルをコンテキストに追加
- `@terminal` → ターミナルの出力（エラーログなど）を渡す
- `@problems` → VSCodeの問題タブの内容を渡す
- `@diff` → 今の変更差分を渡す

### ステップ4：スラッシュコマンドのカスタマイズ

チャット欄で `/` を打つと使えるコマンドです。自分のよく使うプロンプトをコマンド化しておくと効率が跳ね上がります。

```json
{
  "slashCommands": [
    {
      "name": "review",
      "description": "コードレビューをお願いする",
      "prompt": "このコードをレビューしてください。特に①バグの可能性 ②パフォーマンス ③可読性 の観点で指摘してください。日本語で回答してください。"
    },
    {
      "name": "test",
      "description": "テストコードを生成する",
      "prompt": "このコードに対するユニットテストを書いてください。エッジケースも含めて網羅的にお願いします。使用フレームワーク: Vitest"
    },
    {
      "name": "doc",
      "description": "JSDocコメントを追加する",
      "prompt": "このコードにJSDocコメントを追加してください。引数、戻り値、例外、使用例を含めてください。日本語で。"
    }
  ]
}
```

### ステップ5：実際の使い方パターン3選

**パターンA：エラー解析（最頻出）**

ターミナルでエラーが出たら、`@terminal` でエラーをコンテキストに追加し「このエラーの原因と修正方法を教えてください」と聞くだけ。エラーコードをコピペする手間が消えます。

**パターンB：コードレビュー代替**

変更した差分を `@diff` で渡し、`/review` コマンドを実行。PRを出す前のセルフチェックとして使うと、レビュー指摘を事前に潰せます。

**パターンC：既存コードの理解**

レガシーコードを読むとき、対象ファイルを `@file` で追加し「このコードが何をやっているか、わかりやすく説明してください」と聞きます。コメントがないコードでも、ざっくりした地図ができます。

## Continue.dev vs 主要ツール比較表

Continue.devが向いているケース、そうでないケースを整理しておきます。

| 比較軸 | Continue.dev | Cursor | GitHub Copilot |
|---|---|---|---|
| 月額費用 | $0〜（API実費） | $20 | $10〜$19 |
| エディタ | VSCode/JetBrains | 専用IDE | VSCode/各種IDE |
| モデル選択 | 自由（20+モデル） | 固定（Claude/GPT） | 固定（Copilot） |
| インライン補完 | 対応 | 対応 | 対応 |
| チャット機能 | 対応 | 対応 | 対応 |
| カスタマイズ性 | 非常に高い | 中程度 | 低い |
| 初期設定の手軽さ | 中程度 | 簡単 | 簡単 |
| ローカルモデル対応 | 完全対応（Ollama） | 非対応 | 非対応 |
| チーム共有設定 | 対応（`.continue/`） | 部分対応 | 非対応 |

**Continue.devがベストな人：**
- 既存のVSCode環境を変えたくない
- コストを最小化したい
- モデルを自分で選びたい（特定のモデルへのこだわりがある）
- プロジェクトごとに細かくAIの振る舞いをカスタマイズしたい

**Cursorがベストな人：**
- とにかく即座に使い始めたい
- 設定を考えたくない
- VSCode互換で使い勝手を統一したい（Cursorは実質VSCodeフォーク）

**GitHub Copilotがベストな人：**
- 企業のセキュリティポリシー上、データをクラウドに渡したくない（Enterprise版）
- IDE横断で統一ツールを使いたい

他のツールとの詳細な比較は[こちらの記事](/articles/ai-tools-comparison-2024)も参考にしてください。

## まとめ：「無料」で「自由」な選択肢がある

Continue.devを使い始めて気づいたのは、AI開発ツールの選択肢は思ったより広いということです。

有料ツールが悪いわけじゃない。でも「高いから高品質」「無料だから妥協」という前提は、もう成り立ちません。

自分のワークフローに合わせてモデルを選び、コンテキストの渡し方を最適化し、スラッシュコマンドで定型作業を自動化する——この3つを組み合わせたContinue.devは、私にとって「最強のAI IDE」になりました。
